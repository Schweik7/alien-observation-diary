# 地球人观察日记 · 在线多人版

实时多人猜测游戏：选一个外星观察员代号 → 创建房间 → 把房间号/链接发给家人朋友 → 一起猜猜彼此。

基于主规则的「合作猜测」机制：每轮一人当**样本**（秘密选出真实反应），其余人当**观察员**（秘密猜样本会怎样），同时翻牌，猜中越多，全家的**理解值**越高。

## 技术栈
- 后端：Node + Express + Socket.IO（房间与实时同步，状态在内存中）
- 前端：Vite + React + socket.io-client
- 牌库：复用项目的三套 CSV（高年级 / 低年级 / 夫妻版），服务端启动时解析

## 运行

```bash
cd game-app
pnpm install
```

**开发模式**（前后端分开，前端热更新）：
```bash
pnpm dev
# 前端 http://localhost:5174 （在浏览器打开这个）
# 后端 socket 服务 http://localhost:4001
```

**生产/局域网模式**（一个端口同时托管前端和后端）：
```bash
pnpm build      # 构建前端到 ../dist
pnpm start      # 访问 http://localhost:4001
```

## 怎么和别人一起玩
- 同一台机器 / 局域网：让别人访问 `http://<你的IP>:4001`（先 `pnpm build && pnpm start`），或开发模式下访问 `http://<你的IP>:5174`。
- 公网邀请：用 `ngrok http 4001`（或 cloudflared）把本机端口暴露成公网链接，把链接发给对方即可。
- 进入后选好代号 → 创建房间 → 顶部「复制邀请链接」发给对方；对方打开链接会自动填好房间号。

## 玩法说明
- 支持 **2–8 人**。每轮样本顺序自动轮换。
- 抽到**事件卡** → 玩法 A：**表/里两层**——
  - **表层**：样本先选"表面会怎么表现"（🔴炸毛/🔵讲理/🟢求抱/🟡躲开）。
  - **里层**：再选"心里其实最想要的"。观察员只猜**里层**（表面都看得见，难的是底下那层）。
  - 里层词汇随牌库切换：**夫妻版**用一套 12 张「需要/感受」（想被在乎/想被认可/想被理解/想要空间/怕失去你/觉得不公平/我有点愧疚/我就是累了/我也慌/想更亲近你/夹在中间/其实心里是暖的）；**孩子牌库**沿用四色。
- 抽到**反应卡** → 玩法 B：从配套场景卡里选「哪张真的最会让我这样」。
- **每人一句话**：选完后样本要写下"心里那句真话"（必填），观察员可写下"为什么这么猜"（选填）。翻牌时一起亮出——这句话才是真正的引擎。
- **观察日记**：每局把各轮的"表→里 + 那句话"攒成一篇日记，结算页可回看，尤其标出「表里不一」的几条。
- 计分（按人数缩放）：
  - 目标理解值 = `5 + 3 ×（观察员人数）`
  - 失败上限（无人猜中累计）= `玩家数 + 1`
  - 玩法 A 比的是**里层**是否猜中；每位猜中 +1；全员猜中再 +1；无人猜中 −1 且计一次失败。
- 🔴 红卡（掏心窝）任何人可在出牌阶段点「跳过」。
- 翻牌后会提示「先好奇、别辩解」——这是这个游戏真正的意义。

## 目录结构
```
game-app/
  server/
    index.js     Express + Socket.IO 入口，生产环境托管 dist
    game.js      房间与游戏状态机（轮换、计分、胜负、结算）
    decks.js     CSV 解析 + 把卡编译成"可玩单元"（玩法A/玩法B）
    decks/*.csv  三套牌库副本
  client/
    src/
      App.jsx          socket 订阅 + 路由（首页/观测室/对局/结算）
      net.js           socket 连接
      nicknames.js     预设观察员代号
      screens/         Home / Lobby / GameBoard / GameOver
      styles.css       外星观测主题样式
  vite.config.js  root=client，构建到 ../dist
```
